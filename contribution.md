## 🎨 Coding Guidelines

We aim to keep the codebase clean, readable, and consistent.

### General

* Use functional components and React hooks
* Prefer TypeScript (if enabled)
* Keep components small and reusable
* Avoid unnecessary complexity

### Naming

* Components: `PascalCase`
* Variables & functions: `camelCase`
* Constants: `UPPER_CASE`

### Styling

* Follow existing styling patterns (CSS Modules, Tailwind, etc.)
* Avoid inline styles unless necessary

### File Structure

* Group related files together
* Keep components in their own folders if complex

## 🧾 Commit Guidelines

Use clear and descriptive commit messages.

**Format:**

```
type(scope): short description
```

**Examples:**

* `feat(auth): add login form validation`
* `fix(button): resolve disabled state bug`
* `docs(readme): update installation steps`

**Common Types:**

* `feat` – new feature
* `fix` – bug fix
* `docs` – documentation
* `refactor` – code improvement
* `test` – adding tests
* `chore` – maintenance

## 🔀 Pull Request Process

1. Ensure your branch is up to date with `main`
2. Run linting and tests before submitting
3. Provide a clear description of your changes
4. Link related issues (if applicable)
5. Add screenshots or recordings for UI changes
6. Keep PRs small and focused

Once submitted:

* A maintainer will review your PR
* Feedback may be requested — please be responsive
* After approval, your PR will be merged 🎉
